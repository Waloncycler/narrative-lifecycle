# 13 Decision Records

## DR-001: Stage First, Score Second

Scores are auxiliary. Stage gates prevent high scores from bypassing missing evidence.

## DR-002: Add Pricing Adoption in v0.2

S4/S5 boundary was unclear without a dedicated pricing module. Pricing Adoption separates "can trade" from "can allocate".

## DR-003: Separate Parent Reality and Branch Reality

BCI showed that branch reality can be strong while parent reality remains early.

## DR-004: Data Confidence Must Cap Judgment

Insufficient data must lower confidence and cap stages, not be treated as negative evidence.

## DR-005: Add Narrative Memory

Old topics often reappear. The system must compare current evidence with historical state and compute Narrative Delta.

## DR-006: MVP Is Semi-automatic

The first version should validate theory and rules before full automation.
