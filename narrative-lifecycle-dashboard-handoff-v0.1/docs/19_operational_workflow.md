# 19 Operational Workflow

## Daily Workflow

1. Add or ingest new events.
2. Match to Topic Library and Narrative Memory.
3. Extract evidence nodes.
4. Mark affected topics/branches dirty.
5. Update relevant cards only.

## Weekly Workflow

1. Review Early Radar candidates.
2. Update stage changes.
3. Add new branches.
4. Identify failure signals.
5. Generate Weekly Narrative Brief.

## Monthly Workflow

1. Review successful transitions.
2. Review failed candidates.
3. Add failure cases.
4. Update scoring rules if needed.
5. Record rule and prompt versions.

## Human Review

Manual overrides must be structured records, not informal notes.

Every override requires:

- target
- field changed
- old value
- new value
- reason
- reviewer
- rule version
